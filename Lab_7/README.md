## Завдання 11 (варіант 5) — "дерево" (шляхи викликів), що призводять до системних викликів програми, без використання `strace`

**Мета:**  
Отримати список системних викликів, що виконує програма, разом з трасою (user-space stack trace) функцій, які призвели до ць��го виклику. Для кожного системного виклику показується стек викликів у користувацькому просторі.

**Рішення (рекомендований спосіб — bpftrace / eBPF):**  
`bpftrace` дозволяє відслідковувати tracepoint-ові події `syscalls` та друкувати користувацький стек під час входу в системний виклик. Це працює без `strace`, дає дуже детальні дані і вимагає прав `root`.

Приклад однорядкової команди (потрібні права root):
```bash
sudo bpftrace -e 'tracepoint:syscalls:sys_enter_* /comm == "l7z11"/ { printf("\n--- %s %d %s ---\n", comm, pid, probe); ustack(); }' -c './l7z11 arg1 arg2'
```

Пояснення:
- `tracepoint:syscalls:sys_enter_*` — підписка на подію входу в усі системні виклики.
- `/comm == "l7z11"/` — фільтр по імені виконуваного файлу, щоб дивитися тільки нашу програму (замініть на реальну назву, або видаліть фільтр, щоб бачити всі).
- `printf(...)` — друк заголовка з іменем процесу, PID та ім'ям проби (syscall).
- `ustack()` — друк стека користувацького простору, тобто шляхів викликів у процесі, які привели до цього системного виклику.
- `-c './l7z11 arg1 arg2'` — опція `bpftrace`, яка запускає цільову програму і проводить трасування її виконання.

- Для трасування по PID можна використати фільтр `/pid == 12345/`.
- Якщо треба лише підрахувати або звести статистику по syscalls, можна збирати агрегації, наприклад:
  ```bash
  sudo bpftrace -e 'tracepoint:syscalls:sys_enter_* { @[probe] = count(); }' -c './l7z11'
  ```
  і зупинити `Ctrl-C` для виведення агрегованих результатів.
- Альтернативи: `perf` (`perf trace` / `perf record` + `perf script`) або BCC інструменти (наприклад, `execsnoop`, `opensnoop`) — вони теж не є `strace` і можуть бути використані в залежності від потреб.

Запуск:
1. Компілюємо з відлагодженням:
   ```
   gcc -g l7z11.c -o l7z11
   ```
2. Запускаємо трасування:
   ```
   sudo bpftrace -e 'tracepoint:syscalls:sys_enter_* /comm == "l7z11"/ { printf("\n--- %s %d %s ---\n", comm, pid, probe); ustack(); }' -c './l7z11'
   ```

Це дасть послідовність записів: для кожного входу в системний виклик буде надрукований стек викликів у користувацькому просторі — таким чином ви отримаєте інформацію про "дерево" викликів, що призвело до кожного `syscall`.
