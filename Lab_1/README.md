```
light@vm:~$ gcc --version                                                   
gcc (Ubuntu 13.3.0-6ubuntu2~24.04.1) 13.3.0                                              
```
Це означає що компілятор готовий працювати.

## Завдання 1.5

## Мета роботи
Ознайомитися з принципами модульної компіляції в мові C та визначити, які файли потрібно перекомпілювати при зміні різних частин проєкту.

## Структура проєкту

Проєкт складається з наступних файлів:

main.c       → stdio.h, process1.h
input.c      → stdio.h, list.h
output.c     → stdio.h
process1.c   → stdio.h, process1.h
process2.c   → stdio.h, list.h
utility.c    → stdio.h, process1.h

---

## Залежності файлів

Файл main.c залежить від process1.h
Файл input.c залежить від list.h
Файл output.c не має додаткових залежностей
Файл process1.c залежить від process1.h
Файл process2.c залежить від list.h
Файл utility.c залежить від process1.h

---

## Перекомпіляція файлів

### Зміна у process1.c

Файл process1.c містить лише реалізацію, тому його зміна не впливає на інші файли.

Потрібно перекомпілювати:
process1.c

### Зміна у process1.h

Цей заголовочний файл використовується в інших модулях, тому зміна впливає на всі файли, які його підключають.

Потрібно перекомпілювати:
main.c
process1.c
utility.c

### Зміна у list.h

Цей заголовочний файл використовується у відповідних модулях.

Потрібно перекомпілювати:
input.c
process2.c

## Додавання нового файлу

Було додано файл utility.c, який залежить від stdio.h та process1.h.

Це означає, що при зміні process1.h файл utility.c також потрібно перекомпілювати.


## Висновки

Файли з розширенням .c містять реалізацію і при зміні впливають тільки на себе.
Файли з розширенням .h містять інтерфейс і впливають на всі файли, які їх використовують.
Зміни в заголовочних файлах викликають більшу кількість перекомпіляцій.
Правильна організація залежностей дозволяє зменшити час збірки програми.


## Kомпіляція
```
gcc -c main.c                                                           
gcc -c input.c                                           
gcc -c output.c                                          
gcc -c process1.c                                     
gcc -c process2.c                                    
gcc -c utility.c                                         
```
```
gcc main.o input.o output.o process1.o process2.o utility.o -o program
```
