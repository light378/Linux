# Варіант 5
## Тема роботи
Реалізувати витік пам’яті, який виникає тільки при помилці системного виклику і не проявляється у “успішному” сценарії

## Суть помилки
У програмі виділяється динамічна пам’ять (буфер), після чого виконується системний виклик open(). Якщо файл вдається відкрити – пам’ять звільняється (free). Якщо виклик повертає помилку (файл не існує або немає прав доступу) – програма завершується достроково, не викликаючи free. Таким чином, витік виникає лише в гілці обробки помилки, а в нормальному сценарії ресурс звільняється коректно.

## Результати запуску
Успішний сценарій (файл `data.txt` існує та доступний для читання)
```
$ gcc l5z5.c -o l5z5
$ ./l5z5 data.txt
Success: file opened, memory freed correctly.      
```
## Перевірка через valgrind підтверджує відсутність витоків:
```
==1001== HEAP SUMMARY:
==1001==     in use at exit: 0 bytes in 0 blocks
==1001==     total heap usage: 2 allocs, 2 frees, 1,048,576 bytes allocated
==1001== All heap blocks were freed -- no leaks are possible
```
## Сценарій із помилкою системного виклику (файл missing.txt не існує)
```
$ gcc l5z5.c -o l5z5
$ ./l5z5 missing.txt
Error opening file: No such file or directory
Memory leak occurs (1 MB not freed).
```

```

==1002==     total heap usage: 2 allocs, 1 frees, 1,048,576 bytes allocated
```

## Висновок
Витік пам’яті виникає лише за наявності помилки системного виклику (open повертає -1).
В успішному випадку всі ресурси звільняються, що повністю відповідає поставленій умові завдання.
